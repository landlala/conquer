# Project Overview

## Goal

참고 이미지와 동일한 모양의 한반도 퍼즐 웹 페이지를 제작한다.

웹 페이지에는 한반도 지도를 크게 표시하고, 지도는 참고 이미지와 같은 방식으로 20개의 조각으로 분리되어야 한다.

---

# Reference Image

반드시 제공된 참고 이미지 `korea.png`를 기준으로 작업한다.

구현 시 다음 요소를 참고 이미지와 최대한 동일하게 맞춘다.

- 한반도 전체 외곽선
- 제주도 위치와 모양
- 울릉도/독도 위치와 모양
- 내부 퍼즐 조각 분리선
- 각 조각의 번호 위치
- 전체적인 비율과 배치

---

# Map Requirements

## Shape

- 한반도 모양은 참고 이미지의 형태를 그대로 사용한다.
- 임의의 다른 한반도 지도 이미지를 사용하지 않는다.
- 격자 형태로 새로 나누지 않는다.
- 참고 이미지에 있는 퍼즐 분리 방식을 그대로 사용한다.

## Puzzle Pieces

- 전체 지도는 총 20개의 조각으로 구성한다.
- 내륙 조각은 1번부터 18번까지이다.
- 제주도는 19번 조각이다.
- 울릉도와 독도는 합쳐서 20번 조각으로 취급한다.
- 울릉도와 독도 위에는 숫자 20을 각각 따로 쓰지 말고, 두 섬 위쪽 또는 근처에 한 번만 표시한다.

---

# Initial State

초기 화면에서는 다음 상태를 만족해야 한다.

- 모든 퍼즐 조각은 동일한 연한 회색으로 표시한다.
- 각 조각 사이의 경계선은 보이도록 한다.
- 외곽선은 내부 경계선보다 더 굵고 진하게 표시한다.
- 내부 경계선은 참고 이미지처럼 점선 또는 얇은 선으로 표시한다.
- 각 조각에는 1부터 20까지의 번호가 표시된다.
- 번호 위치는 참고 이미지와 최대한 동일하게 배치한다.

---

# Interaction

사용자가 각 퍼즐 조각을 클릭하면 다음 동작을 수행한다.

## Color Change

- 클릭한 조각만 색상이 변경된다.
- 조각마다 서로 다른 색상을 사용한다.
- 이미 클릭된 조각은 다시 클릭할 수 없다.

## Number Change

클릭한 조각의 기존 번호를 새로운 랜덤 숫자로 변경한다.

랜덤 숫자는 다음 조건을 만족해야 한다.

- 10의 배수
- 200 이하
- 가능한 값: 10, 20, 30, ..., 200

---

# Technical Requirements

- HTML, CSS, JavaScript로 구현한다.
- 가능하면 SVG를 사용하여 각 퍼즐 조각을 독립적인 `<path>`로 만든다.
- 각 조각은 개별적으로 클릭 이벤트를 가져야 한다.
- 참고 이미지 위에 단순히 색상 레이어를 덮는 방식이 아니라, 실제로 각 조각이 독립적으로 선택 가능해야 한다.
- 코드 구조는 가독성 있게 작성한다.

---

# Expected Behavior

초기 화면:

- 참고 이미지와 같은 한반도 퍼즐 지도가 표시된다.
- 총 20개의 조각이 존재한다.
- 모든 조각은 연한 회색이다.
- 번호는 1부터 20까지 표시된다.

클릭 후:

- 클릭한 조각의 색상이 다른 색으로 변경된다.
- 클릭한 조각의 숫자가 10부터 200 사이의 10의 배수 중 하나로 변경된다.
- 클릭하지 않은 조각은 그대로 유지된다.

---

# Important Notes

- 가장 중요한 것은 참고 이미지의 모양과 분할 방식을 그대로 따르는 것이다.
- 한반도 외곽선과 내부 분리선이 참고 이미지와 크게 달라지면 안 된다.
- 단순한 랜덤 분할이나 격자 분할은 사용하지 않는다.
- 제주도는 19번, 울릉도와 독도는 합쳐서 20번으로 처리한다.
